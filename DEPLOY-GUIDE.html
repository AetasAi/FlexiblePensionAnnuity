3

sitemap.xml — Tell Google about every page

No sitemap currently exists. Without it, newly added pages may not be discovered by Google for weeks.

~5 mins Add 1 file to repo root Submit URL to Search Console
Step 1 — Add sitemap.xml to the repository root

The sitemap.xml file is in your deliverables. Add it to the root of the repository alongside index.html. No other configuration needed — GitHub Pages serves it automatically.

Step 2 — Submit to Google Search Console
  • Go to search.google.com/search-console
  • Add property: https://aetasai.github.io/FlexiblePensionAnnuity/
  • Verify ownership via the HTML meta tag method (paste one tag into index.html <head>)
  • Once verified, go to Sitemaps → Submit: sitemap.xml
  • Search Console will now show indexing status, crawl errors, and keyword positions
Important — Update sitemap.xml when you add new pages

Every time a new page is added (e.g. a new insight article), add it to sitemap.xml with today's date as the <lastmod> value. This tells Google to re-crawl promptly.

Microsoft Clarity — Free heatmaps and session recordings

Currently no visual behaviour data. Clarity is free, takes 5 minutes to install, and shows you exactly where advisers click, scroll, and leave — far more actionable than GA4 alone.

~10 mins Free — no cost Paste 1 script into all pages
Step 1 — Create a free Clarity account
  • Go to clarity.microsoft.com
  • Sign in with a Microsoft account (or create one free)
  • Click "Add new project" → enter site name and URL
  • Copy the project ID (a short alphanumeric string, e.g. abc123xyz)
Step 2 — Paste the Clarity script into all pages

Replace YOUR_CLARITY_PROJECT_ID with your actual project ID from Step 1.

HTML — paste into <head> of all pages
<!-- Microsoft Clarity — heatmaps and session recordings -->
<script type="text/javascript">
    (function(c,l,a,r,i,t,y){
        c[a]=c[a]||function(){(c[a].q=c[a].q||[]).push(arguments)};
        t=l.createElement(r);t.async=1;t.src="https://www.clarity.ms/tag/"+i;
        y=l.getElementsByTagName(r)[0];y.parentNode.insertBefore(t,y);
    })(window, document, "clarity", "script", "YOUR_CLARITY_PROJECT_ID");
</script>
Add to all pages — place immediately after the GA4 script block index.html insight-double-whammy.html insight-bpr-legal-basis.html insight-executor-liability.html

Data appears in the Clarity dashboard within 2 hours of the first session. After 1 week you'll have heatmaps showing exactly where advisers click on the calculator and where they drop off.

Step 2 — Add tab deep-linking to index.html

The nav links above store the target tab name in sessionStorage before navigating to index.html. Add this small script to index.html — paste it just before the closing </body> tag. It reads the stored tab name on load and opens that tab automatically.

JavaScript — add to index.html before </body>
// Deep-link from insight pages — open the correct tab on arrival
(function() {
  var tab = sessionStorage.getItem('openTab');
  if (tab) {
    sessionStorage.removeItem('openTab');
    // showTab is the existing function in index.html
    if (typeof showTab === 'function') {
      showTab(tab);
      // Scroll the nav tab into view
      var tabEl = document.querySelector('.ntab[data-tab="' + tab + '"');
      if (tabEl) tabEl.scrollIntoView({behavior: 'smooth', block: 'nearest'});
    }
  }
})();
Note on tab IDs — verify the exact tab IDs used in the existing showTab() calls in index.html. They appear to be: overview, problem, solution, how-it-works, case-studies, costs, calculator, insights, faqs. Adjust the sessionStorage values in the nav links above to match exactly.
